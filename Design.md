# 学生助教管理平台系统设计文档

## 1. 项目概述

学生助教管理平台是一个面向高校的在线管理系统，旨在简化和规范助教招募、工作跟踪与薪酬核算的全流程。系统通过角色权限控制，为不同用户（学生、助教、教师、管理员）提供定制化功能，实现助教管理的数字化、自动化与透明化。

## 2. 用户角色与权限设计

### 2.1 学生（Student）

- **权限**：
  - 浏览助教岗位
  - 投递简历（应聘）
  - 查看申请状态
  - 修改个人信息

### 2.2 助教（Teaching Assistant - TA）

- **说明**：助教是“学生”角色的特殊状态，申请通过后自动转换。
- **权限**：
  - 继承学生所有权限
  - 填写并提交工作日志/工时
  - 查看薪酬记录

### 2.3 教师（Faculty / Department）

- **权限**：
  - 发布助教招募需求
  - 管理已发布岗位
  - 筛选和审核学生简历
  - 录用/拒绝申请者
  - 审批助教工作日志/工时
  - 评价助教工作

### 2.4 系统管理员（Administrator）

- **权限**：
  - 管理所有用户账号（增删改查）
  - 查看全平台数据统计（岗位数、申请数、在岗人数、薪酬支出等）
  - 生成月度薪酬报表

## 3. 核心功能模块设计

### 3.1 统一身份认证模块

- 用户注册、登录、个人信息管理
- 单点登录（SSO）支持
- 基于角色的权限控制（RBAC）

### 3.2 招募管理模块

- **教师端**：创建/编辑/关闭岗位（含课程、职责、人数、薪资、截止日期等）
- **学生端**：浏览/搜索/筛选/排序开放岗位，查看详情

### 3.3 应聘流程模块

- **学生端**：在线填写/上传简历，投递岗位，查看申请进度（已投递 → 教师查看 → 待录用 → 已录用/已拒绝）
- **教师端**：查看申请列表，预览简历，标记状态，发送录用/拒绝通知

### 3.4 工作跟踪与核算模块

- **助教端**：按月填写工作日志，提交工时表
- **教师端**：审核工时表（批准/驳回），评价助教工作

### 3.5 通知与消息模块

- 自动发送关键节点通知（站内信/邮件）
- 包括：新岗位发布、申请状态变更、工时审核提醒、薪酬发放等

### 3.6 数据看板模块

- **管理员端**：全校宏观数据展示（岗位数、申请数、在岗人数、薪酬支出等）
- **教师端**：个人岗位状态、申请人数、助教工作情况等

## 4. 技术架构与选型

### 4.1 技术栈

- **前端**：Vue.js（界面美观、上手快）
- **后端**：Python + Django（框架成熟、开发效率高）
- **数据库**：MySQL（稳定、易管理）
- **开发工具**：VSCode（前后端开发）、Navicat（数据库管理）
- **版本控制**：Git + GitHub

### 4.2 部署方案

- **平台选择**：PythonAnywhere（专为Python项目打造的托管平台）
- **优势**：
  - 支持用户访问、数据库、静态文件、任务调度（如APScheduler）
  - 提供免费MySQL数据库（适合毕设规模）
- **限制与应对**：
  - **用户文件存储**：需使用外部存储服务（如AWS S3、阿里云OSS等）保存简历等文件
  - **域名**：仅支持子域名（如yourusername.pythonanywhere.com）
  - **资源限制**：免费账户有CPU/内存限制，但足够毕设演示使用
  - **外部API调用**：需提前配置允许访问的API地址

## 6. 系统创新与安全设计

### 6.1 敏感信息保密

- 采用多层次、纵深防御的数据安全体系
- 用户密码不可逆存储（哈希加密）
- 基于角色的精细权限控制（RBAC）
- 二重机制保障数据安全与隐私

### 6.2 流程自动化

- 关键节点自动通知（如申请状态变更、工时提交、薪酬发放等）
- 实现方式：
  - 数据库创建通知表，事件触发后自动插入待发送记录
  - 独立Worker脚本定时扫描通知表，完成站内信/邮件推送
